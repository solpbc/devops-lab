Ran `run.sh` Part 2 on the `Standard_DC2as_v5` CVM. Steps 6–8 all passed on real
hardware:

fixture to version 1 to match hardware. Self-test still 10/10. (`e3b1d7c`)

The freshness loop is now **closed and demonstrated on real hardware**: AMD
report → runtime-data binding → AMD-bound AK → fresh guest-bound quote, no MAA.

### 3. Toy verifier (`verify.sh`)
Added an in-container **toy verifier** to make the attester/verifier split
concrete (the real, off-CVM verifier remains the next milestone).
`verify.sh challenge` issues a nonce; `run.sh` binds it (now persists it to
`nonce.hex`); `verify.sh appraise` independently re-runs the checks — HCLA
header, runtime-data binding, AMD chain (optional pinned ARK), AK↔`HCLAkPub`,
and the quote whose qualifying data must equal the binding recomputed from the
verifier's *own* nonce — and only on all-pass wraps a stand-in LUKS key to the
guest X25519 pubkey (ECDH → SHA-256 KDF → AES-CTR) and proves the guest can
unwrap it. Toy gaps flagged inline (`[TOY GAP]`): same host as attester,
fetched-ARK trust unless pinned, unauthenticated AES-CTR, record-then-pin PCR
policy. Verified off-hardware: tampered `report_data` refuses to release;
`test/verifier-selftest.sh` covers policy parsing + key-release round-trip
(5/5), wired into `build-check.sh` Tier 3. (`943cc81`)

### 4. One-command demo (`demo.sh`)
Wrapped the three stages into a single orchestrator and made it the container's
default `ENTRYPOINT`, so a bare `podman run … solpbc` on the CVM runs the whole
story: STAGE 1 [verifier] issues a nonce → STAGE 2 [attester] `run.sh` binds it
and produces evidence → STAGE 3 [verifier] `verify.sh appraise` checks it and
releases the toy key. `run.sh`/`verify.sh` stay individually runnable via
`--entrypoint`. **Validated on hardware**: ran the bare `podman run … solpbc`
and the full flow worked end to end, all stage-3 checks passing. Off-hardware
the no-TPM guard prints guidance and exits 0. (`d97d723`)

**Milestone reached:** the composite Azure SEV-SNP attestation — AMD report,
runtime-data binding, AMD-bound vTPM AK, fresh guest-bound quote, and key
release — is implemented, demonstrated on real hardware in a single command,
and covered off-hardware by self-tests, with no MAA in the trust path.
