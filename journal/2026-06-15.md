# Journal ŌĆö 2026-06-15

## Session 3 ŌĆö vTPM quote freshness binding

Picked up the milestone flagged at the end of [2026-06-13](2026-06-13.md): the
freshness half of the design. The AMD report half (ARK ŌåÆ ASK ŌåÆ VCEK ŌåÆ report
signature) was already done and demonstrated on hardware; this session built
and tested the rest of the composite evidence path.

### 1. `lib/hcl.sh` ŌĆö the custom logic beyond `snpguest`
New library of pure-shell helpers (deps: `openssl`, `jq`, `xxd`, `base64`, `dd`,
`grep`, `awk` ŌĆö all already in the runtime container, no python):
- `hcl_verify_header` ŌĆö check the 32-byte HCLA header (sig `HCLA`, version 2, request_type 2).
- `hcl_amd_report` / `hcl_runtime_json` ŌĆö split the embedded 1184-byte AMD report and the runtime-data JSON out of the blob. The JSON extractor anchors on the first `{"` at/after offset 1216 and runs to the first NUL (JSON has no NUL; Azure NUL-pads the NV region), so it doesn't depend on a fixed offset for the variable-data metadata.
- `hcl_verify_runtime_binding` ŌĆö confirm `SHA-256(runtime data) == report_data[0..32]`. This both proves the binding **and** confirms the JSON byte range was parsed exactly.
- `hcl_akpub_modulus_hex` / `tpm_akpub_modulus_hex` / `hcl_verify_ak_binding` ŌĆö prove the live vTPM AK is the AMD-bound `HCLAkPub` by matching RSA moduli. **Design choice:** comparing moduli avoids reconstructing a PEM from the JWK `n`/`e`; the quote is still verified under the TPM's own AK PEM, and the modulus match is what ties that PEM to the AMD report.
- `hcl_binding_hash` ŌĆö `H(domain Ōłź nonce Ōłź guest_pubkey Ōłź ctx)` for the quote qualifying data.

### 2. `run.sh` extended to the full composite flow
Restructured into Part 1 (steps 1ŌĆō4, AMD report ŌĆö unchanged) and **Part 2
(steps 5ŌĆō8, freshness binding)**: read the HCLA blob (`tpm2_nvread` NV
`0x01400001`), verify the runtime-data binding, prove the AK binding
(`tpm2_readpublic` `0x81000003`), then take and verify an AK-signed quote
(`tpm2_quote`/`tpm2_checkquote`) over the PCRs with qualifying data
`H("sol-key-release-v1" Ōłź nonce Ōłź guest_pubkey Ōłź ctx)`. Everything is
env-overridable (`PCR_LIST`, `NONCE_HEX`, `AK_HANDLE`, `HCL_NV_INDEX`,
`BINDING_DOMAIN`, `CTX_FILE`). Off-hardware it still exits cleanly.

### 3. Off-hardware test ŌĆö `test/freshness-selftest.sh`
There's no TPM in the dev sandbox, so the `tpm2_*` quote step can't run without
a CVM. Everything around it can: the test fabricates a synthetic HCLA blob (a
stand-in RSA AK, a runtime JSON carrying it as `HCLAkPub`, and an AMD report
whose `report_data` is `H(runtime data)`, plus 20 bytes of binary metadata and
NUL padding to mimic the real layout) and drives every `lib/hcl.sh` function.
**10/10 pass**, including negative cases (tampered `report_data`, wrong AK, bad
HCLA signature all rejected). Wired in as Tier 3 (`selftest`) of
`test/build-check.sh`.

### 4. Containerfile + README
`Containerfile` now `COPY`s `lib/` into the image. README updated: repo layout,
quick-start note about the self-test, and a new "What `run.sh` implements"
subsection walking the eight steps.
