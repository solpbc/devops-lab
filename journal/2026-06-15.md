# Journal — 2026-06-15

## Session 3 — vTPM quote freshness binding

Picked up the milestone flagged at the end of [2026-06-13](2026-06-13.md): the
freshness half of the design. The AMD report half (ARK → ASK → VCEK → report
signature) was already done and demonstrated on hardware; this session built
and tested the rest of the composite evidence path.

### 1. `lib/hcl.sh` — the custom logic beyond `snpguest`
New library of pure-shell helpers (deps: `openssl`, `jq`, `xxd`, `base64`, `dd`,
`grep`, `awk` — all already in the runtime container, no python):
- `hcl_verify_header` — check the 32-byte HCLA header (sig `HCLA`, version 2, request_type 2).
- `hcl_amd_report` / `hcl_runtime_json` — split the embedded 1184-byte AMD report and the runtime-data JSON out of the blob. The JSON extractor anchors on the first `{"` at/after offset 1216 and runs to the first NUL (JSON has no NUL; Azure NUL-pads the NV region), so it doesn't depend on a fixed offset for the variable-data metadata.
- `hcl_verify_runtime_binding` — confirm `SHA-256(runtime data) == report_data[0..32]`. This both proves the binding **and** confirms the JSON byte range was parsed exactly.
- `hcl_akpub_modulus_hex` / `tpm_akpub_modulus_hex` / `hcl_verify_ak_binding` — prove the live vTPM AK is the AMD-bound `HCLAkPub` by matching RSA moduli. **Design choice:** comparing moduli avoids reconstructing a PEM from the JWK `n`/`e`; the quote is still verified under the TPM's own AK PEM, and the modulus match is what ties that PEM to the AMD report.
- `hcl_binding_hash` — `H(domain ∥ nonce ∥ guest_pubkey ∥ ctx)` for the quote qualifying data.

### 2. `run.sh` extended to the full composite flow
Restructured into Part 1 (steps 1–4, AMD report — unchanged) and **Part 2
(steps 5–8, freshness binding)**: read the HCLA blob (`tpm2_nvread` NV
`0x01400001`), verify the runtime-data binding, prove the AK binding
(`tpm2_readpublic` `0x81000003`), then take and verify an AK-signed quote
(`tpm2_quote`/`tpm2_checkquote`) over the PCRs with qualifying data
`H("sol-key-release-v1" ∥ nonce ∥ guest_pubkey ∥ ctx)`. Everything is
env-overridable (`PCR_LIST`, `NONCE_HEX`, `AK_HANDLE`, `HCL_NV_INDEX`,
`BINDING_DOMAIN`, `CTX_FILE`). Off-hardware it still exits cleanly.

### 3. Off-hardware test — `test/freshness-selftest.sh`
There's no TPM in the dev sandbox, so the `tpm2_*` quote step can't run without
a CVM. Everything around it can: the test fabricates a synthetic HCLA blob (a
stand-in RSA AK, a runtime JSON carrying it as `HCLAkPub`, and an AMD report
whose `report_data` is `H(runtime data)`, plus 20 bytes of binary metadata and
NUL padding to mimic the real layout) and drives every `lib/hcl.sh` function.
**10/10 pass**, including negative cases (tampered `report_data`, wrong AK, bad
HCLA signature all rejected). Wired in as Tier 3 (`selftest`) of
`test/build-check.sh`.

### 4. Containerfile + README
`Containerfile` now `COPY`s `lib/` into the image. README updated: repo layout,
quick-start note about the self-test, and a new "What `run.sh` implements"
subsection walking the eight steps.

### Status
The composite Azure SEV-SNP evidence path is now **implemented end to end** and

### 5. Validated on a live CVM ✅ (same day)
Ran Part 2 on the `Standard_DC2as_v5` CVM. Steps 6–8 all passed on real
hardware:
- **report_data == H(runtime data)** matched (`cbc6216a…`) — the runtime-data
  byte-range extraction (the one part the self-test could only approximate) is
  correct against a real Azure blob. Extracted 1110 bytes of JSON claims.
- **vTPM AK == AMD-bound HCLAkPub** matched (512 hex chars = RSA-2048).
- **Fresh AK-signed quote** over `sha256:0,2,4,7,8,9,15,16,22,23` verified;
  qualifying data matched the binding hash.

**One fix from the run:** the live HCLA header reports **version 1**, not the 2
the brief claimed. The header check was too strict and printed a (non-fatal)
warning while everything downstream still verified. Relaxed `hcl_verify_header`
to accept version 1 or 2 and key on `request_type==2`; updated the self-test
fixture to version 1 to match hardware. Self-test still 10/10.

The freshness loop is now **closed and demonstrated on real hardware**: AMD
report → runtime-data binding → AMD-bound AK → fresh guest-bound quote, with no
MAA in the trust path.
